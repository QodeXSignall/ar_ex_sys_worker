Ошибки, логирование, отладка
============================

Логирование отправок
--------------------

Все отправки фиксируются в `ex_sys_data_send_reports`:
- `sent` — факт отправки,
- `get` — подтверждение успешного ответа,
- `ex_sys_data_id` — идентификатор во внешней системе,
- `data` — сериализованный payload.

Это позволяет:
- повторно отправлять только неуспешные записи,
- отслеживать ошибки по внешнему ID,
- чистить логи по месту (например, при удалении актов).

Частые проблемы
---------------

1) Нет данных для отправки  
Причина: фильтры по времени/категориям или записи уже отправлены.  
Решение: проверить `time_start`, `trash_cats_*` и `ex_sys_data_send_reports`.

2) Ошибка авторизации  
Причина: неверные логин/пароль или токен.  
Решение: проверить `core_settings` и метод `auth()`.

3) Нет фото/видео  
Причина: отсутствует запись в `photos`/`videos`.  
Решение: проверить пути, наличие файлов и права.

4) Ошибка формата данных  
Причина: несовпадение полей с API внешней системы.  
Решение: проверить `format_send_data` конкретного воркера.

Отладочные рекомендации
-----------------------

- Включить логгер и печать ответов (в коде много `print`).
- Проверить SQL‑запросы из `mixins.ActsSQLCommands`.
- Для Signall есть тестовый стенд: `set_signall_test()`.
